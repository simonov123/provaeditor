INFORMATIVA PRIVACY   (IT)
1)Accesso ai dati della cartella dei Documenti Pubblici

Installando Provaeditor l’utente autorizza l’applicazione all’accesso alla cartella “C:/Users/Public/Documents” sia in lettura sia in scrittura dei file

1.1-il file lang.txt

è il file dove è memorizzata la lingua dell’editor di testo (Italiano,Inglese o Russo)
viene creato al momento dell’impostazione della lingua (al primo avvio di Provaeditor) e viene salvato nella cartella dei documenti pubblici.
Avviene poi l’accesso in lettura ogni volta che viene avviata l’app per caricare la lingua.

-L’utente autorizza Provaeditor a creare il file lang.txt e ad accederci.

-Se il file lang.txt esiste già nella cartella dei documenti pubblici da prima dell’installazione di Provaeditor,l’applicazione ci accederà in lettura (senza modificare,salvare o condividere nulla) e l’editor procederà nella lingua di default,ovvero l’Inglese (almenochè il file lang presistente non abbia al suo interno le stringhe “Italiano” o “Русский”,ad esempio nel caso di una precedente installazione di Provaeditor)

1.2-i file CSV di salvataggio dei font

l’utente autorizza Provaeditor a salvare nella cartella dei Documenti Pubblici un file contenente le informazioni sul font del file in formato .csv

ad esempio,se viene creato il file “nome.txt”,nella cartella dei documenti pubblici viene creato il file “nome.txt.csv” contenente il nome e le dimensioni del font da ricaricare alla riapertura del file.

La creazione del file avviene insieme al salvataggio del file creato dall’editor,e a ogni accesso a un file avviene l’accesso in lettura e il caricamento del suo font (se esiste)

per tanto l’utente autorizza Provaeditor alla creazione e alla lettura dei file .csv

 si sconsiglia di salvare nei documenti pubblici file .csv che finiscono in .txt (nome.txt.csv),a differenza del file lang,a differenza del file lang questi file se già persenti potrebbero essere sovrascritti,e se letti (e non conformi) potrebbero creare problemi al programma.





1.3)i file ricevuti con la funzione di sharing

Provaeditor integra una funzione di file-sharing che permette di condividere il file in editing purché sia grande massimo 80KB.

La condivisione avviene solo all’interno della stessa rete ma non è protetta.
I file non vengono crittografati

Provaeditor si solleva da qualsiasi responsabilità legata a problemi di sicurezza della connessione

I file vengono salvati nella cartella dei documenti pubblici,ma il nome del file ricevuto viene scelto dall’utente.

L’Utente autorizza provaeditor a scrivere il file ricevuto.
E tiene presente che in caso di file con lo stesso nome verrà sovrascritto.



2)Installazione della JRE
L’utente autorizza il Setup di Provaeditor a installare la JRE di Oracle,componente fondamentale per il funzionamento del programma in quanto è scritto nel linguaggio Java.

3)Scrittura e lettura file
Essendo un’Editor di testo l’utente autorizza Provaeditor ad accedere in lettura e scrittura,e a sovrascrivere file in caso di omonimia dei file,nei limiti dei privilegi che ha l’Utente (Amministratore o Limitato.)


PRIVACY DISCLAIMER   (EN)
1) Access to data in the Public Documents folder

By installing Provaeditor the user authorizes the application to access the "C:/Users/Public/Documents" folder both for reading and writing files

1.1 - the lang.txt file

is the file where the text editor language is stored (Italian, English or Russian)
it is created when the language is set (the first time Provaeditor is started) and is saved in the public documents folder.
Read access then occurs every time the app is started to load the language.

-The user authorizes Provaeditor to create the lang.txt file and access it.

-If the lang.txt file already exists in the public documents folder before installing Provaeditor, the application will access it for reading (without modifying, saving or sharing anything) and the editor will proceed in the default language, i.e. 'English (unless the existing lang file does not have the strings “Italiano” or “Русский” inside it, for example in the case of a previous installation of Provaeditor)

1.2 - CSV files for saving fonts

the user authorizes Provaeditor to save a file containing information on the font of the file in .csv format in the Public Documents folder

for example, if the file "name.txt" is created, the file "name.txt.csv" is created in the public documents folder containing the name and size of the font to be reloaded when the file is reopened.

The creation of the file occurs together with the saving of the file created by the editor, and each time a file is accessed, reading access and loading of its font (if it exists) occurs.

therefore the user authorizes Provaeditor to create and read .csv files

 it is not recommended to save other external .csv files that end in .txt (name.txt.csv) in public documents, unlike the lang file, unlike the lang file these files if already missing could be overwritten, and if read (and non-compliant) could create problems for the program.





1.3) files received with the sharing function

Provaeditor integrates a file-sharing function that allows you to share the file being edited as long as it is a maximum size of 80KB.

Sharing occurs only within the same network but is not protected.
The files are not encrypted

Provaeditor relieves itself of any responsibility related to connection security problems

The files are saved in the public documents folder, but the name of the received file is chosen by the user.

The User authorizes provaeditor to write the received file.
And keep in mind that in case of files with the same name it will be overwritten.



2)Installation of the JRE
The user authorizes Provaeditor Setup to install the Oracle JRE, a fundamental component for the operation of the program as it is written in the Java language.

3) Writing and reading files
Being a text editor, the user authorizes Provaeditor to access reading and writing, and to overwrite files in case of homonymy of the files, within the limits of the privileges that the User has (Administrator or Limited.)




ОТКАЗ ОТ ОТВЕТСТВЕННОСТИ О КОНФИДЕНЦИАЛЬНОСТИ   (RU)
1) Доступ к данным в папке Public Documents

Устанавливая Provaeditor, пользователь разрешает приложению доступ к папке «C:/Users/Public/Documents» как для чтения, так и для записи файлов.

1.1 - файл lang.txt

— файл, в котором хранится язык текстового редактора (итальянский, английский или русский)
он создается при установке языка (при первом запуске Provaeditor) и сохраняется в папке общедоступных документов.
Доступ для чтения происходит каждый раз, когда приложение запускается для загрузки языка.

-Пользователь разрешает Provaeditor создать файл lang.txt и получить к нему доступ.

-Если файл lang.txt уже существует в папке общедоступных документов до установки Provaeditor, приложение получит доступ к нему для чтения (без изменения, сохранения или совместного использования чего-либо), и редактор будет работать на языке по умолчанию, т.е. «английском» (если не установлен в существующем языковом файле нет строк «Italiano» или «Русский», например, в случае предыдущей установки Provaeditor)

1.2 — CSV-файлы для сохранения шрифтов

пользователь разрешает Provaeditor сохранить файл, содержащий информацию о шрифте файла в формате .csv, в папке Public Documents

например, если создается файл «имя.txt», в папке общедоступных документов создается файл «имя.txt.csv», содержащий имя и размер шрифта, который будет перезагружаться при повторном открытии файла.

Создание файла происходит вместе с сохранением файла, созданного редактором, и при каждом обращении к файлу происходит доступ на чтение и загрузка его шрифта (если он существует).

поэтому пользователь разрешает Provaeditor создавать и читать файлы .csv.

не рекомендуется сохранять файлы .csv, оканчивающиеся на .txt (имя.txt.csv), в общедоступных документах, в отличие от файла lang, в отличие от файла lang, эти файлы, если они уже отсутствуют, могут быть перезаписаны, и если оно прочитано (и не соответствует требованиям), это может создать проблемы для программы.
1.3) файлы, полученные с помощью функции обмена

В Provaeditor интегрирована функция обмена файлами, которая позволяет вам делиться редактируемым файлом, если его максимальный размер составляет 80 КБ.

Совместное использование происходит только внутри одной сети, но не защищено.
Файлы не зашифрованы

Provaeditor снимает с себя любую ответственность, связанную с проблемами безопасности соединения.

Файлы сохраняются в папке общедоступных документов, но имя получаемого файла выбирает пользователь.

Пользователь уполномочивает провайдера записать полученный файл.
И имейте в виду, что в случае файлов с таким же именем оно будет перезаписано.

Пользователь разрешает программе установки Provaeditor установить Oracle JRE, основной компонент для работы программы, написанной на языке Java.
